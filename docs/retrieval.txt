第一层：structured filter / graph filter
先用你已有的结构条件缩小范围，比如：
	•	只看某个 group_id
	•	只看 Entity
	•	只看 RELATES_TO
	•	只看 relation_family = EMPLOYMENT

这个阶段不是为了“找最像”，而是为了先把无关数据排掉。

第二层：lexical / full-text search
再用 name、summary、fact、content 做关键词召回。
这层适合查：
	•	明确出现过的词
	•	专有名词
	•	缩写
	•	关系词
	•	精确短语

比如你搜 OpenAI, CEO, GraphRAG，full-text 往往很好用。

第三层：embedding semantic search
最后才是 embedding 去找语义上相近但字面不完全一样的内容。

比如：
	•	没写 works at，但写了 is employed by
	•	没写 founded，但写了 co-created
	•	summary 里是近义表达，不是原词

这时候 embedding 很有价值。


方案 A：先找 entity，再扩 edge
	1.	用 OpenAI 去找最像的 entity node
	2.	找到 canonical entity uuid
	3.	再查和这个 entity 相连的 edges

用户query
-> 识别核心实体（OpenAI）
-> 找到 canonical entity / aliases
-> 用 entity uuid + alias expansion 缩小范围
-> 再搜相关 edges/facts/episodes
-> rerank


query -> entity resolution -> edge retrieval 的最小可用流程。

这层查的通常是：
	•	entity.name_embedding
	•	entity.summary_embedding
	•	edge.fact_embedding
	•	episode.content_embedding
